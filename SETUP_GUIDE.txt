================================================================================
  InsightAgent — Full Setup Guide (Step by Step)
================================================================================

This guide walks you through setting up the InsightAgent project from scratch
on a local Windows machine, getting it running, and optionally deploying to
the cloud.

================================================================================
  PREREQUISITES
================================================================================

Before you begin, make sure you have the following installed:

  1. Python 3.10 or 3.11
     - Download: https://www.python.org/downloads/
     - During install, check "Add Python to PATH"
     - Verify: py --version

  2. Git
     - Download: https://git-scm.com/downloads
     - Verify: git --version

  3. Docker Desktop (for the database and optional containerized deployment)
     - Download: https://www.docker.com/products/docker-desktop/
     - Make sure Docker Engine is running before starting services
     - Verify: docker --version && docker compose version

  4. (Optional) An OpenAI API key or compatible LLM API key
     - Required for real LLM insight generation
     - You can use LLM_ADAPTER=mock for testing without an API key

  5. (Optional) A News API key from https://newsapi.org
     - Required if NEWS_API_ENABLED=true
     - You can disable it with NEWS_API_ENABLED=false


================================================================================
  STEP 1 — Clone the Repository
================================================================================

  git clone <your-repo-url> InsightAgent
  cd InsightAgent


================================================================================
  STEP 2 — Create a Python Virtual Environment
================================================================================

  py -m venv venv

Activate the virtual environment:

  Windows (PowerShell):
    .\venv\Scripts\Activate.ps1

  Windows (CMD):
    venv\Scripts\activate.bat

  Git Bash / WSL:
    source venv/Scripts/activate

You should see (venv) in your terminal prompt.


================================================================================
  STEP 3 — Install Dependencies
================================================================================

For a full local install (core + optional features):

  pip install --upgrade pip
  pip install -r requirements.txt

For a minimal install (core only, no Streamlit/LLM/ML):

  pip install -r requirements-core.txt

For development tools (pytest, alembic):

  pip install -r requirements-dev.txt

Summary of requirement files:
  - requirements-core.txt     FastAPI, SQLAlchemy, psycopg, Pydantic, etc.
  - requirements-optional.txt Streamlit, Pandas, LangGraph, OpenAI, scikit-learn
  - requirements.txt          core + optional combined
  - requirements-dev.txt      pytest, alembic


================================================================================
  STEP 4 — Create Your .env File
================================================================================

Copy the example environment file:

  Windows (PowerShell):
    Copy-Item .env.example .env

  Git Bash / Linux:
    cp .env.example .env

Open .env in your editor and configure the following values:

  REQUIRED SETTINGS:
  ------------------
  APP_MODE=cloud
    (The app requires APP_MODE=cloud to start — this is by design)

  DATABASE_URL=postgresql+psycopg://insight:local_dev_change_me_123@localhost:5433/insight_agent
    (Use localhost if running the API outside Docker but PostgreSQL inside Docker)
    (Use postgres instead of localhost if running everything inside Docker)

  POSTGRES_PASSWORD=local_dev_change_me_123
    (Must match the password in DATABASE_URL)

  LLM SETTINGS:
  ------------------
  For real LLM usage:
    LLM_API_KEY=sk-your-openai-api-key-here
    (or set OPENAI_API_KEY=sk-your-openai-api-key-here)

  For testing without an LLM:
    LLM_ADAPTER=mock
    (This returns a fixed mock response — no API key needed)

  NEWS API:
  ------------------
  If you have a News API key:
    NEWS_API_KEY=your_news_api_key_here

  If you don't have one, disable the connector:
    NEWS_API_ENABLED=false


================================================================================
  STEP 5 — Start PostgreSQL with Docker Compose
================================================================================

Start the database container:

  docker compose up postgres -d

This starts a PostgreSQL 16 instance on port 5433.

Verify it's running:

  docker compose ps

You should see insight_postgres with status "healthy".

(Optional) Start PgAdmin for a database GUI:

  docker compose --profile dev-tools up pgadmin -d

  Then open http://localhost:5050
    Email:    admin@local.dev
    Password: local_dev_change_me_123


================================================================================
  STEP 6 — Run Database Migrations
================================================================================

With your virtual environment active and .env configured:

  py -m alembic upgrade head

This creates all required tables in the database.

Verify migrations applied:

  py -m alembic current
  py -m alembic heads

Both should show the same revision hash = your DB is up to date.


================================================================================
  STEP 7 — Start the FastAPI Backend
================================================================================

Run the API server:

  py -m uvicorn app.main:app --host 0.0.0.0 --port 8000 --reload

The API is now live at: http://localhost:8000

Verify it's working:

  Open http://localhost:8000/health in your browser
  or
  curl http://localhost:8000/health

You should see a JSON response like:
  {
    "insight": "Service healthy",
    "evidence": "API and scheduler lifecycle initialized successfully.",
    ...
  }

API docs (auto-generated by FastAPI):
  Swagger UI:  http://localhost:8000/docs
  ReDoc:       http://localhost:8000/redoc


================================================================================
  STEP 8 — Start the Streamlit Frontend (Optional)
================================================================================

In a separate terminal (with your venv activated):

  py -m streamlit run streamlit_app.py

This opens the Streamlit dashboard in your browser at http://localhost:8501

Note: The Streamlit app requires the FastAPI backend to be running.
Make sure you have the optional dependencies installed (requirements.txt
or requirements-optional.txt).


================================================================================
  STEP 9 — Run Tests
================================================================================

With your venv active:

  py -m pytest tests/ -v

This runs the full test suite.


================================================================================
  RUNNING EVERYTHING WITH DOCKER COMPOSE (Alternative to Steps 5-7)
================================================================================

Instead of running the API natively, you can run the entire stack in Docker:

  docker compose up --build -d

This starts:
  - PostgreSQL (insight_postgres) on port 5433
  - FastAPI backend (insight_backend) on port 8000

The backend waits for PostgreSQL to be healthy before starting.

View logs:
  docker compose logs -f backend

Stop everything:
  docker compose down

To also remove database data:
  docker compose down -v

================================================================================
  ENVIRONMENT VARIABLE REFERENCE (Key Variables)
================================================================================

  APP_MODE                  Must be "cloud" (required)
  DATABASE_URL              PostgreSQL connection string (required)
  LLM_ADAPTER               "openai" (default) or "mock"
  LLM_API_KEY               Your LLM API key (or use OPENAI_API_KEY)
  NEWS_API_ENABLED          true/false — toggles News API connector
  NEWS_API_KEY              Your newsapi.org key
  GOOGLE_TRENDS_ENABLED     true/false — toggles Google Trends connector
  WORLD_BANK_ENABLED        true/false — toggles World Bank connector
  UPLOAD_STORAGE_DIR        Path for CSV uploads (default: data/uploads)
  ENVIRONMENT               "local" or "production"
  WEB_CONCURRENCY           Number of uvicorn workers (default: 1)
  SQL_ECHO                  true/false — log SQL queries
  POSTGRES_DB               Database name (default: insight_agent)
  POSTGRES_USER             Database user (default: insight)
  POSTGRES_PASSWORD         Database password (required for Docker)


================================================================================
  PRODUCTION DEPLOYMENT (Cloud Mode)
================================================================================

  1. Copy .env.production.example to .env.production and fill in real values
  2. Build the Docker image:
       docker build -t insight-agent:1.0.0 .
  3. Push to your container registry
  4. Deploy using docker-compose.prod.yml:
       docker compose -f docker-compose.prod.yml up -d
  5. Run migrations against the production DB:
       ALEMBIC_DATABASE_URL=<prod-db-url> py -m alembic upgrade head


================================================================================
  TROUBLESHOOTING
================================================================================

  Problem: "APP_MODE is not set" error on startup
  Solution: Add APP_MODE=cloud to your .env file

  Problem: "No database URL configured" error
  Solution: Set DATABASE_URL in your .env file

  Problem: "LLM API key is not set" error
  Solution: Set LLM_API_KEY or OPENAI_API_KEY, or use LLM_ADAPTER=mock

  Problem: "NEWS_API_KEY is not set" error
  Solution: Set NEWS_API_KEY or set NEWS_API_ENABLED=false

  Problem: Database connection refused
  Solution: Make sure PostgreSQL is running (docker compose up postgres -d)
            Use localhost in DATABASE_URL when running the API outside Docker

  Problem: "Schema mismatch" error on startup
  Solution: Run migrations: py -m alembic upgrade head

  Problem: Alembic can't connect to database
  Solution: Make sure DATABASE_URL is set in your .env and PostgreSQL is running

  Problem: Import errors for optional packages (streamlit, openai, etc.)
  Solution: Install full deps: pip install -r requirements.txt
